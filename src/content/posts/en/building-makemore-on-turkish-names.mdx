The plan is to work through the fundamentals the hard way: a character-level
language model trained from scratch on Turkish names, then deployed so it runs
**entirely in your browser**. No inference server, no per-request cost.

Below is the model, live. Press **generate** to sample a name one character at a
time. Drag **temp** down for safer, more common names; up for stranger ones.

<ModelDemo name="turkish-makemore" client:visible />

Right now this is a placeholder, a character-level *bigram* counted from a small
list of names, just enough to prove the pipeline end to end. The real writeup will
cover the interesting part: Turkish vowel harmony, and whether a tiny model can
learn it from names alone.

## The pipeline

Training happens in a notebook (PyTorch). A single script turns the trained model
into a static artifact (weights plus a `meta.json` describing the vocab and
hyperparameters) and drops it into `public/models/`. The forward pass and sampling
then run in a few dozen lines of plain TypeScript, right here on the page.

That is the whole thesis of this site in miniature: **train elsewhere, deploy
here, run in the reader's browser.**
